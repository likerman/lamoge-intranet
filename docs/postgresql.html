Guía de acceso y uso de PostgreSQL

Este documento reúne la información básica para solicitar acceso, conectarse a las bases de datos y registrar parámetros operativos relevantes. Sustituya los valores de ejemplo por la información real del entorno.

Datos de conexión

  • Servidor: completar con hostname o IP.
  • Puerto: completar si difiere del puerto estándar.
  • Base de datos: indicar nombre de la base asignada.
  • Usuario: especificar usuario o rol provisto.

Acceso desde línea de comandos

Use un bloque como referencia para documentar la forma recomendada de conexión:

psql -h HOST -p PUERTO -U USUARIO -d BASE_DE_DATOS

Buenas prácticas

  • Registrar quién administra cada base y el propósito de cada esquema.
  • Indicar políticas de respaldo y retención de datos.
  • Documentar restricciones de red y procedimientos de recuperación.
