Guía de acceso por SSH

Esta guía reúne el procedimiento recomendado para acceder al servidor de LaMoGe desde Windows o Linux, organizar el trabajo dentro de la cuenta asignada y utilizar de forma correcta el espacio de respaldo.

Datos necesarios antes del primer acceso

  • Dirección IP o nombre del servidor.
  • Nombre de usuario asignado por administración.
  • Contraseña inicial o mecanismo de acceso definido para la cuenta.
  • Ruta de la carpeta de trabajo y de la carpeta privada de backup.
Servidor: 157.92.29.231
Usuario: usuario
Acceso SSH: ssh usuario@157.92.29.231
Carpeta de trabajo: /datos/usuario
Carpeta de backup: /backup/users/usuario

Primer acceso desde Windows

Windows 10 y Windows 11 incluyen cliente SSH, por lo que no hace falta instalar software adicional para conectarse por terminal.

  1. Abrir PowerShell o Símbolo del sistema.
  2. Ejecutar el comando de conexión provisto por administración.
  3. Confirmar la huella del servidor la primera vez escribiendo yes.
  4. Ingresar la contraseña cuando sea solicitada. Aunque no se vean caracteres en pantalla, el sistema sí la está recibiendo.
  5. Si el sistema lo pide, cambiar la contraseña inicial en el primer acceso.
ssh usuario@IP_DEL_SERVIDOR

# Ejemplo
ssh ftapia@157.92.29.231

Acceso desde Linux

En Linux el acceso por SSH se realiza del mismo modo, desde una terminal estándar del sistema.

ssh usuario@IP_DEL_SERVIDOR

# Ejemplo
ssh ftapia@157.92.29.231
  • Si aparece un pedido de confirmación de la huella, revisar que el servidor sea el esperado y confirmar solo si corresponde.
  • Si la cuenta fue entregada con contraseña inicial, es recomendable cambiarla apenas se complete el primer ingreso.
  • Cuando se definan claves públicas o archivos de configuración SSH institucionales, esta guía debería actualizarse con ese procedimiento.

Cambio de contraseña

Si la cuenta usa contraseña, el cambio puede realizarse dentro del servidor con el siguiente comando:

passwd

El sistema pedirá la contraseña actual y luego la nueva contraseña dos veces. Se recomienda una longitud mínima de 12 caracteres con combinación de mayúsculas, minúsculas, números y símbolos.

Espacios disponibles para cada usuario

Cada cuenta dispone de dos ubicaciones principales con funciones distintas.

  • /datos/usuario: espacio de trabajo habitual para proyectos activos, scripts, resultados y datos de uso frecuente.
  • /backup/users/usuario: espacio privado de respaldo para copias de seguridad, versiones archivadas y resguardo de información importante.
/datos/ftapia
/backup/users/ftapia

Comandos básicos dentro del servidor

pwd                  # ver carpeta actual
ls -lah              # listar archivos con detalle
cd                   # volver al directorio personal
mkdir nombre_carpeta # crear carpeta
cd nombre_carpeta    # entrar a una carpeta
cd ..                # subir un nivel
touch archivo.txt    # crear archivo vacío
rm archivo.txt       # eliminar archivo
rmdir carpeta        # eliminar carpeta vacía
rm -r carpeta        # eliminar carpeta con contenido

El comando rm -r debe usarse con cuidado, especialmente fuera del espacio personal de trabajo.

Verificaciones mínimas después del acceso

Una vez que la cuenta fue creada o entregada, conviene validar que el acceso y las rutas principales funcionen correctamente.

pwd
ls -lah
mkdir prueba_acceso
rm -r prueba_acceso
mkdir -p /backup/users/usuario/prueba
ls -lah /backup/users/usuario

Transferencia de archivos desde Windows

Para subir o bajar archivos desde una PC con Windows se recomienda WinSCP mediante SFTP.

  • Protocolo: SFTP
  • Host name: IP o nombre del servidor
  • Puerto: 22
  • User name: nombre de usuario asignado
  • Password: contraseña correspondiente

Las rutas de uso más frecuentes son /datos/usuario para trabajo activo y /backup/users/usuario para respaldos.

Buenas prácticas de organización

Se recomienda mantener una estructura simple y consistente dentro de cada cuenta.

/datos/usuario/proyectos/
/datos/usuario/scripts/
/datos/usuario/datos/
/datos/usuario/resultados/
/datos/usuario/tmp/

/backup/users/usuario/pc/
/backup/users/usuario/home_servidor/
/backup/users/usuario/archivo/
  • Usar nombres de archivo descriptivos y breves.
  • Evitar espacios y acentos en nombres de carpetas o archivos.
  • Agregar fechas cuando ayuden a identificar versiones o respaldos.
  • No mezclar trabajo activo con copias de seguridad.

Errores comunes

  • Permission denied: suele indicar usuario, contraseña o permisos incorrectos, o intento de escritura en una ruta no autorizada.
  • Connection refused: puede señalar un problema de red, una IP incorrecta o un servicio no disponible.
  • Host key verification failed: consultar con administración antes de eliminar huellas previas o continuar.
  • Si la contraseña no se ve al escribir, el comportamiento es normal en terminales SSH.

Seguridad operativa básica

  • No compartir credenciales ni claves privadas con otras personas.
  • Cambiar la contraseña inicial en el primer acceso si aplica.
  • Trabajar habitualmente dentro de /datos/usuario.
  • Usar /backup/users/usuario solo para respaldos y resguardo.
  • Realizar backups periódicos de trabajos importantes y de la computadora personal.
