Guía de inicio

Usos del servidor

Esta guía está pensada para usuarios que recién comienzan a trabajar con el servidor de LaMoGe y necesitan entender para qué sirve, cómo ingresar y cómo usarlo de manera ordenada y segura.

¿Para qué sirve este servidor?

El servidor concentra infraestructura compartida del laboratorio para tareas como:

  • ejecutar servicios técnicos como JupyterLab, WebODM, GeoServer o bases de datos
  • trabajar con proyectos y datos desde una cuenta institucional
  • guardar resultados y materiales de uso frecuente
  • mantener copias de seguridad y archivo de información importante

Qué necesita un usuario para empezar

  • usuario asignado por administración
  • contraseña inicial o credenciales de acceso
  • dirección IP o nombre del servidor
  • saber cuál es su carpeta de trabajo y su carpeta de backup
Carpeta de trabajo: /datos/usuario
Carpeta de backup: /backup/users/usuario

Cómo ingresar al sistema

La forma más habitual es conectarse por SSH desde una terminal.

ssh usuario@IP_DEL_SERVIDOR
  • en Windows se puede usar PowerShell o Símbolo del sistema
  • en Linux se usa una terminal estándar
  • la primera vez puede pedirse confirmar la huella del servidor
  • la contraseña no se ve al escribir: eso es normal

Para el procedimiento completo, vea la guía de acceso por SSH.

Dónde conviene trabajar

El espacio principal de trabajo es /datos/usuario. Allí conviene guardar:

  • scripts
  • proyectos activos
  • resultados en curso
  • datos que se usan habitualmente

Una estructura simple ayuda mucho a mantener el orden:

/datos/usuario/proyectos/
/datos/usuario/scripts/
/datos/usuario/datos/
/datos/usuario/resultados/
/datos/usuario/tmp/

Por qué hacer backups

Un backup no es una copia decorativa: sirve para reducir pérdidas de trabajo ante errores humanos, borrados accidentales, cambios no deseados o problemas de hardware.

Conviene respaldar especialmente:

  • documentos importantes
  • scripts y configuraciones relevantes
  • material de tesis, artículos y planillas
  • datos de campo, resultados consolidados y exportaciones de bases de datos

Cómo hacer backups de forma simple

El espacio pensado para resguardo es /backup/users/usuario. No debería usarse como carpeta de trabajo diario.

Una práctica básica y recomendable es crear subcarpetas claras y con fecha:

/backup/users/usuario/pc/
/backup/users/usuario/home_servidor/
/backup/users/usuario/archivo/
/backup/users/usuario/laptop_2026-04-18/
  • copiar solo material importante o estable
  • evitar cachés, descargas innecesarias y archivos temporales
  • no mezclar trabajo activo con archivo o resguardo
  • revisar periódicamente que los respaldos sean legibles y estén bien ubicados

Qué prácticas ayudan a sacarle provecho al servidor

  • mantener una estructura simple de carpetas
  • usar nombres cortos y descriptivos
  • evitar espacios y acentos en archivos y directorios
  • separar claramente proyectos activos, resultados y respaldos
  • limpiar archivos temporales o duplicados cuando ya no hagan falta
  • consultar la documentación antes de usar servicios nuevos o rutas sensibles
